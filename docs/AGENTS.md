# Documentation

Contribution and reference documentation for the Omarchy dotfiles project.

## Structure

- `contributing/` — Git workflow, commit conventions, PR guidelines
- `reference/` — Package documentation, keybinding reference

## Critical

**Read `contributing/CONTRIBUTING.md` before making PRs.**

Commit conventions: `Add` | `Fix` | `Update` | `Remove` | `docs:` | `chore:`

## Boundaries

- **Always:** Update docs when changing related code
- **Always:** Follow commit conventions in `CONTRIBUTING.md`
- **Never:** Let docs drift from implementation
